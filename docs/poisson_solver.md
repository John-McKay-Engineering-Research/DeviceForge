# Poisson Solver

## Overview

DeviceForge includes a two-dimensional finite-difference solver for the electrostatic Poisson equation.

Unlike the Laplace equation, which assumes no internal charge, the Poisson equation models electrostatic potential generated by fixed charge within semiconductor materials.

The governing equation is:

[
\nabla^2 \phi = -\frac{\rho}{\varepsilon}
]

where:

* (\phi) is the electrostatic potential (V)
* (\rho) is the volumetric charge density (C/m³)
* (\varepsilon) is the absolute permittivity (F/m)

The current implementation uses a Jacobi iterative solver and serves as the foundation for future semiconductor device simulation capabilities.

---

# Numerical Formulation

For a uniform Cartesian grid with spacing (h), the Poisson equation is discretised using the finite-difference method.

The update equation for each interior grid point is

[
\phi_{i,j}^{k+1}
================

\frac{1}{4}
\left(
\phi_{i+1,j}^{k}
+
\phi_{i-1,j}^{k}
+
\phi_{i,j+1}^{k}
+
\phi_{i,j-1}^{k}
+
h^2
\frac{\rho_{i,j}}
{\varepsilon_{i,j}}
\right)
]

where

* (k) denotes the iteration number
* (h) is the grid spacing
* (\rho) is the fixed charge density
* (\varepsilon) is the material permittivity

This update is evaluated repeatedly until the specified convergence tolerance is reached.

---

# Fixed Charge Model

The current DeviceForge implementation assumes fully ionised fixed dopants and neglects mobile carriers.

The fixed charge density is therefore

[
\rho
====

q
\left(
N_D
---

N_A
\right)
]

where

* (q) is the elementary charge
* (N_D) is donor concentration
* (N_A) is acceptor concentration

Positive values correspond to net donor charge.

Negative values correspond to net acceptor charge.

This approximation is appropriate for validating the numerical implementation before introducing self-consistent semiconductor physics.

---

# Material Permittivity

The absolute permittivity is calculated using

[
\varepsilon
===========

\varepsilon_0
\varepsilon_r
]

where

* (\varepsilon_0) is the vacuum permittivity
* (\varepsilon_r) is the relative permittivity of the material

The current Poisson solver assumes spatially constant permittivity throughout the computational domain.

Support for multiple dielectric materials will be added in a future release.

---

# Boundary Conditions

The current solver supports:

## Dirichlet Boundary Conditions

Fixed electrostatic potential

[
\phi = V_{\mathrm{specified}}
]

Typical examples include electrical contacts and grounded conductors.

---

## Homogeneous Neumann Boundary Conditions

Zero normal electric field

[
\frac{\partial\phi}{\partial n}=0
]

These represent electrically insulated boundaries.

Only homogeneous (zero-gradient) Neumann conditions are currently supported.

---

# Solver Workflow

The Poisson solver performs the following operations:

1. Validate the simulation configuration.
2. Construct the fixed charge-density field.
3. Construct the absolute permittivity field.
4. Compute the electrostatic source term.
5. Initialise the potential field.
6. Perform Jacobi iterations.
7. Apply Neumann boundary conditions.
8. Reapply Dirichlet boundary conditions.
9. Calculate the maximum potential change.
10. Check convergence.
11. Return the completed simulation result.

---

# Validation

The initial validation problem consists of a uniformly doped silicon region with:

* constant donor concentration
* constant permittivity
* zero potential at both ends
* insulated upper and lower boundaries

The analytical solution is

[
\phi(x)
=======

\frac{\rho}{2\varepsilon}
x(L-x)
]

which forms a parabola across the device.

The numerical solution is validated by comparing the centre-line potential against this analytical solution.

---

# Electric Field

Once the electrostatic potential has converged, the electric field is calculated using

[
\mathbf{E}
==========

*

\nabla\phi
]

where

* (E_x = -\partial\phi/\partial x)
* (E_y = -\partial\phi/\partial y)

DeviceForge computes the electric field as a post-processing step using finite differences.

---

# Outputs

The solver returns:

* Electrostatic potential
* Fixed charge density
* Absolute permittivity
* Electrostatic source term
* Residual history
* Runtime
* Iteration count
* Convergence status
* Solver metadata

These quantities are stored within the `SimulationResult` object for subsequent visualisation and analysis.

---

# Current Limitations

The present implementation assumes:

* Two-dimensional Cartesian grids
* Uniform grid spacing
* Spatially constant permittivity
* Fully ionised dopants
* Fixed charge only
* Linear Poisson equation
* Jacobi iteration
* NumPy execution

---

# Future Development

The Poisson solver provides the numerical foundation for DeviceForge's future semiconductor device capabilities.

Planned developments include:

* Variable-permittivity discretisation
* Piecewise dielectric materials
* Piecewise doping profiles
* PN junction electrostatics
* MOS capacitor simulation
* MOSFET electrostatics
* Nonlinear equilibrium Poisson solver
* Mobile electron and hole concentrations
* Drift-diffusion transport
* Sparse linear solvers
* Multigrid acceleration
* C++ solver backends
* GPU acceleration
* Distributed-memory execution
* Three-dimensional simulation

The Poisson solver represents the transition from a general finite-difference framework to a semiconductor device simulator.

